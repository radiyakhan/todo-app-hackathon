# Frontend UI Skill – Pages, Components & Styling

## Instructions

1. **Page Structure**
   - Semantic HTML layout
   - Clear section hierarchy
   - Reusable components (header, footer, cards, forms)

2. **Layout & Responsiveness**
   - Flexbox and Grid for layout
   - Mobile-first design
   - Responsive breakpoints for tablet and desktop

3. **Styling**
   - Clean and consistent spacing
   - Modern typography
   - Color system with variables
   - Component-based CSS (CSS Modules / Tailwind / BEM)

4. **Interactivity**
   - Hover and focus states
   - Button and link feedback
   - Simple UI animations and transitions

## Best Practices

- Keep components small and reusable
- Maintain consistent spacing and alignment
- Use semantic HTML for accessibility
- Ensure cross-browser compatibility
- Optimize for performance and readability

## Example Structure

```html
<main class="container">
  <header class="navbar">
    <h1 class="logo">App Name</h1>
    <nav class="nav-links">
      <a href="#">Home</a>
      <a href="#">Features</a>
      <a href="#">Contact</a>
    </nav>
  </header>

  <section class="content">
    <div class="card">
      <h2>Component Title</h2>
      <p>Reusable component content</p>
      <button class="primary-btn">Action</button>
    </div>
  </section>

  <footer class="footer">
    <p>© 2026 All rights reserved</p>
  </footer>
</main>
